# Employee Service

NestJS microservice that manages **employee profile data** and **HR admin operations**. It serves both self-service endpoints (profile, photo, password) and admin-only CRUD. It uses the Auth service via gRPC for password changes and user creation, and publishes notification events to RabbitMQ when profile or password changes.

## Features

- **Employee self-service**
  - Get own profile (name, email, photo, position, phone).
  - Update profile: photo (multipart/form-data, uploaded to S3-compatible storage) and phone number.
  - Change password (via Auth service gRPC).
- **Admin HR**
  - Create employee (backend generates UUID; creates user in Auth via gRPC and employee record in DB).
  - List employees with pagination, role filter, and search by name/email.
  - Get employee by ID and update (name, position, phone). Photo is not updatable by admin.
- **Notifications** — On profile or password change, publishes events to RabbitMQ (`EMPLOYEE_UPDATED`, `EMPLOYEE_PASSWORD_CHANGED`) for the Notification service to log and push to admin UI.
- **gRPC server** — Exposes `GetEmployee` for the Attendance service to verify employee existence and active status.

## Architecture

Clean architecture layout:

- **Domain** — `Employee` entity, `EmployeeRepository` interface, role type.
- **Application** — Use cases: `GetMeUseCase`, `UpdateMyProfileUseCase`, `ChangeMyPasswordUseCase`, `CreateEmployeeUseCase`, `ListEmployeesUseCase`, `GetEmployeeByIdUseCase`, `UpdateEmployeeUseCase`.
- **Infrastructure** — PostgreSQL repository, S3 storage service, Auth gRPC client, RabbitMQ publisher.
- **Presentation** — HTTP controllers (employee + admin), DTOs, credentials middleware (reads `x-user-credentials` from Gateway), admin role guard.

Authorization: admin-only routes check for `ADMIN_HR` role; others require a valid user context from the Gateway.

## API (HTTP)

Use base path **`/employee`** when calling via the API Gateway. All routes below require a valid Bearer access token (and Gateway forwards credentials) unless noted.

### Self-service (employee)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/employee/employees/me` | Get current user's profile |
| PATCH | `/employee/employees/me/profile` | Update photo and/or phone (multipart/form-data) |
| PATCH | `/employee/employees/me/password` | Change password (body: `oldPassword`, `newPassword`) |

### Admin (ADMIN_HR only)

| Method | Path | Description |
|--------|------|-------------|
| POST | `/employee/employees/admin` | Create employee (body: name, email, password, position, role?, phone?). `id` is generated by backend. |
| GET | `/employee/employees/admin` | List employees. Query: `page`, `limit`, `role`, `search` |
| GET | `/employee/employees/admin/:id` | Get employee by ID |
| PATCH | `/employee/employees/admin/:id` | Update name, position, phone (not photo) |

Admin endpoints return `403 Forbidden` if the user role is not `ADMIN_HR`.

## gRPC (server)

- **Package:** `employee`
- **Port:** `GRPC_PORT` (default `50020`).

**RPC:** `GetEmployee(GetEmployeeRequest)` → returns `found`, `isActive`, and basic fields (id, name, email, role). Used by Attendance service before recording or listing attendance.

| Variable | Description | Default |
|----------|-------------|---------|
| `PORT` | HTTP port | `3050` |
| `GRPC_PORT` | gRPC server port | `50020` |
| `DB_*` | PostgreSQL (`dexa_attendance`) | — |
| `AUTH_GRPC_URL` | Auth service gRPC address | `localhost:50010` |
| `RABBITMQ_URL`, `NOTIFICATION_QUEUE_NAME` | RabbitMQ for notification events | `notification.events` |
| `S3_ENDPOINT`, `S3_BUCKET`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY` | S3-compatible storage for profile photos | — |
| `LOG_LEVEL` | `DEBUG`, `WARN`, `ERROR` | `DEBUG` |

## Database

- **Database:** `dexa_attendance`.
- **Schema:** `employee`.
- **Tables:** `employees` (id, name, email, position, role, phone_number, photo_url, is_active, timestamps).

Migrations: `migrations/001_employee_up.sql`, `001_employee_down.sql`.

## How to Run

```bash
cd backend/employee-service
npm install
# Configure .env (DB, Auth gRPC, RabbitMQ, S3)
npm run start:dev
```

Or from `backend`: `npm run start:employee`.

Requires: PostgreSQL, Auth service (gRPC), RabbitMQ (for notifications). Optional: S3 for photo uploads (otherwise profile photo update may fail or be skipped depending on implementation).

## Docker

Build image:

```bash
docker build -t dexa-employee-service:latest ./backend/employee-service
```

Run container:

```bash
docker run --rm -p 3050:3050 --env-file ./backend/employee-service/.env dexa-employee-service:latest
```

Notes:

- gRPC server port is configurable via env (`GRPC_PORT`) for Attendance service integration.
- RabbitMQ and S3 endpoints should be reachable from container network.
